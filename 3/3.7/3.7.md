# 3.4. Módulo de Gestión de Reportes
| Requerimiento | Nombre  |
|--------|--------------------------------|
| R024   | Registrar reportes  |
| R025   | Exportar reportes |
| R026   | Programar Envío Automático |
| R027   | Filtrar Reportes |
| R028   | Editar Reportes |
| R029   | Enviar Notificaciones Manualmente |

## 🎯 R024 - Registrar reportes
### 📝 Caso de Uso

| Campo       | Descripcion |
|--------------|------|
| **Nombre** | Registrar Reportes |
| **Tarea realizada** | El usuario ingresa información del reporte (fechas, responsable, área, tipo de evento, etc.) y lo guarda en el sistema. |
| **Actor iniciador** |  |
| **Actor principales** |  |
| **Flujo de eventos** | 1. Usuario hace clic en “Registrar Reportes” <br> 2. Completa los campos requeridos <br> 3. Hace clic en “Agregar Reporte” |
| **Excepciones** | Campos vacíos o fechas inconsistentes impiden guardar el reporte. |
| **Precondicion** | El usuario debe haber iniciado sesión. |
| **Postcondición** | El reporte queda almacenado en el sistema y puede visualizarse. |
| **Casos de uso incluidos** | Validación de campos |
| **Requesito no funcional** | Interfaz clara, validación en tiempo real, respuesta menor a 3 segundos. |

## 🎯 R025 - Exportar reportes
### 📝 Caso de Uso

| Campo       | Descripcion |
|--------------|------|
| **Nombre** |  Exportar reportes |
| **Tarea realizada** | El usuario filtra reportes por fechas, tipo o área y los exporta (posiblemente a Excel o PDF).. |
| **Actor iniciador** | |
| **Actor principales** |  |
| **Flujo de eventos** | 1. Usuario accede a “Exportar Reportes” <br> 2. Aplica filtros <br> 3. Exporta los resultados. |
| **Excepciones** | No hay reportes en el rango de fechas. Error de formato en la exportación. |
| **Precondicion** | Debe haber reportes registrados. |
| **Postcondición** | Se genera un archivo con los reportes seleccionados. |
| **Casos de uso incluidos** |Filtrado de datos |
| **Requesito no funcional** | Formato exportado debe ser compatible con Excel y PDF. Tiempo máximo de exportación: 5 segundos. |

## 🎯 R025 - Programar Envío Automático
### 📝 Caso de Uso

| Campo       | Descripcion |
|--------------|------|
| **Nombre** |  Programar Envío Automático de Reportes |
| **Tarea realizada** | El usuario programa el envío de reportes por correo electrónico en fechas determinadas. |
| **Actor iniciador** |  |
| **Actor principales** |  |
| **Flujo de eventos** | 1. Usuario hace clic en “Programar envío automático” <br> 2. Define destinatarios, frecuencia, filtros <br> 3. Guarda configuración |
| **Excepciones** | Correos inválidos, programación sin reportes disponibles |
| **Precondicion** | Tener reportes cargados y acceso al módulo |
| **Postcondición** | El sistema enviará automáticamente los reportes según programación. |
| **Casos de uso incluidos** | Validación de correo, selección de periodicidad. |
| **Requesito no funcional** | Seguridad en el envío (TLS), disponibilidad 99%, interfaz intuitiva. |

## 🎯 R024 - Filtrar Reportes
### 📝 Caso de Uso

| Campo       | Descripcion |
|--------------|------|
| **Nombre** | Filtrar Reportes |
| **Tarea realizada** | El usuario utiliza filtros para buscar reportes por fechas, tipo de evento, área de trabajo o palabra clave. |
| **Actor iniciador** |  |
| **Actor principales** |  |
| **Flujo de eventos** | 1. Usuario abre la sección “Reportes registrados” <br> 2. Aplica filtros <br> 3. Visualiza resultados filtrados |
| **Excepciones** | No hay resultados que coincidan con los filtros. |
| **Precondicion** | Deben existir reportes registrados. |
| **Postcondición** | Se muestran solo los reportes que cumplen los criterios. |
| **Casos de uso incluidos** | Carga de lista, paginación de resultados |
| **Requesito no funcional** | Búsqueda ágil, respuesta menor a 3 segundos, feedback claro. |

## 🎯 R024 - Editar Reportes
### 📝 Caso de Uso

| Campo       | Descripcion |
|--------------|------|
| **Nombre** | Editar Reportes |
| **Tarea realizada** | El usuario accede a un reporte registrado y actualiza su contenido. |
| **Actor iniciador** |  |
| **Actor principales** |  |
| **Flujo de eventos** | 1. Usuario selecciona un reporte de la tabla. <br> 2. Hace clic en “Editar”. <br> 3. Modifica la información. <br> 4. Hace clic en “Enviar”. |
| **Excepciones** | No seleccionar ningún reporte o ingresar datos inválidos. |
| **Precondicion** | Debe existir al menos un reporte previamente registrado. |
| **Postcondición** | El reporte se actualiza correctamente en el sistema. |
| **Casos de uso incluidos** | Validación de campos, carga de datos previos |
| **Requesito no funcional** | Confirmación visual de guardado, tiempo de respuesta rápido (< 2s). |

## 🎯 R024 - Enviar Notificaciones Manualmente
### 📝 Caso de Uso

| Campo       | Descripcion |
|--------------|------|
| **Nombre** | Enviar Notificaciones Manualmente |
| **Tarea realizada** | El usuario selecciona uno o más reportes y hace clic en “Enviar notificación” para comunicar vía email. |
| **Actor iniciador** |  |
| **Actor principales** |  |
| **Flujo de eventos** | 1. Usuario aplica filtros y selecciona reportes <br> 2. Hace clic en “Enviar notificación” <br> 3. El sistema envía el correo a los responsables |
| **Excepciones** | Reportes sin destinatario definido o falla en envío |
| **Precondicion** | Tener reportes seleccionados con correos válidos |
| **Postcondición** | Los destinatarios reciben la notificación del reporte. |
| **Casos de uso incluidos** | Validación de correo, conexión SMTP. |
